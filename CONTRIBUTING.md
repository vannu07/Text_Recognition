# Contributing to Text_Recognition

Thanks for your interest in contributing! We welcome improvements of all kinds: code, docs, tests, and examples.

Quick steps:

1. Fork the repo and create a branch named `feat/short-description` or `fix/short-description`.
2. Write tests or update existing ones when adding functionality.
3. Keep changes small and focused.
4. Commit with clear messages and push to your fork.
5. Open a PR against `main` with a clear description and link to any related issues.

Code style:
- Follow existing project style (PEP8 for Python).
- Run linters and formatters where applicable.

PR checklist:
- [ ] Branch from latest `main`.
- [ ] Tests added or updated.
- [ ] Documentation updated (if applicable).
- [ ] CI passes (if configured).

If you're new, look for issues in `issues/` labeled `good-first-contribution` or ask maintainers for guidance.
